About SecureAuth

A real-world demonstration of secure authentication using bcrypt, input validation, session management, and 2FA.

Our Mission

SecureAuth was built to demonstrate that proper password security is not complex — it just requires the right tools. Every password goes through bcrypt with a unique cryptographic salt, making brute-force and rainbow table attacks computationally impractical.

🔐

bcrypt Hashing

Passwords hashed at cost factor 10 — 1024 rounds of computation. Makes brute-force attacks extremely slow even with GPU hardware.

🧂

Unique Salts

Each password hash includes a unique 128-bit random salt. Two users with the same password will have entirely different hashes.

📱

Two-Factor Auth

After password verification, a 6-digit TOTP code is required. Prevents account takeover even if passwords are compromised.

🛡️

Input Validation

All fields strictly validated. SQL injection patterns, weak passwords, email format errors, and mismatches are caught before submission.
